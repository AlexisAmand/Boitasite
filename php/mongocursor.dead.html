MongoCursor::dead

(PECL mongo >=0.9.6)

MongoCursor::dead — Vérifie s'il reste des résultats non envoyés depuis la base de données

Description

public MongoCursor::dead ( void ) : bool

La base de données retourne les réponses dans une série de documents, supérieures à 4 MB par documents, par réponse. Cette méthode vérifie si la base de données a encore des séries ou si le jeu de résultats est terminé.

Un curseur devenu "dead" ne signifie pas que MongoCursor::hasNext() retournera FALSE, cela signifie uniquement que la base de données a terminé d'envoyer les résultats au client. Le client doit continuer d'itérer les résultats tant que la méthode MongoCursor::hasNext() ne retourne pas FALSE.

Liste de paramètres

Cette fonction ne contient aucun paramètre.

Valeurs de retour

Retourne TRUE s'il existe d'autres résultats qui n'ont pas encore été envoyés au client, FALSE sinon.
