pg_lo_truncate

(PHP 5 >= 5.6.0, PHP 7)

pg_lo_truncate — Tronque un objet large

Description

pg_lo_truncate ( resource $large_object , int $size ) : bool

pg_lo_truncate() tronque une ressource d'objet large.

Pour utiliser l'interface des objets larges, il est nécessaire de l'enfermer dans un verrou de transaction.

Liste de paramètres

large_object

Ressource d'objet large PostgreSQL (LOB), retourné par pg_lo_open().

size

Le nombre d'octets à tronquer.

Valeurs de retour

Cette fonction retourne TRUE en cas de succès ou FALSE si une erreur survient.

Exemples

Exemple #1 Exemple avec pg_lo_truncate()

<?php
   $doc_oid = 189762345;
   $database = pg_connect("dbname=jacarta");
   pg_query($database, "begin");
   $handle = pg_lo_open($database, $doc_oid, "r");
   // Tronque à 0
   pg_lo_truncate($handle, 0);
   pg_query($database, "commit");
   echo $data;
?>

Historique

Version Description
5.6.0 Ajout de la fonction de troncage. Elle supporte les objets larges 64 bits PostgreSQL 9.3. Le client et le serveur doivent avoir le support de PostgreSQL 9.3 et PHP doit avoir été compilé en 64 bits.
