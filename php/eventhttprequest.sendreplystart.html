EventHttpRequest::sendReplyStart

(PECL event >= 1.4.0-beta)

EventHttpRequest::sendReplyStart — Initialise un bloc de réponse

Description

public EventHttpRequest::sendReplyStart ( int $code , string $reason ) : void

Initialise une réponse qui utilise les en-têtes Transfer-Encoding chunked.

Ceci permet l'appelant à diffuser la réponse au client, et est utile lorsque toutes les données de la réponse ne sont pas immédiatement disponibles, ou bien lors d'envoi de réponses trop volumineuses.

L'appelant doit fournir les blocs de données avec la méthode EventHttpRequest::sendReplyChunk() et compléter la réponse en appelant la méthode EventHttpRequest::sendReplyEnd().

Liste de paramètres

code

Le code réponse HTTP à envoyer.

reason

Un bref message à envoyer avec le code réponse.

Valeurs de retour

Aucune valeur n'est retournée.

Voir aussi

  • EventHttpRequest::sendReplyChunk() - Envoi un autre bloc de données comme partie d'un bloc de réponse entrant
  • EventHttpRequest::sendReplyEnd() - Complète un bloc de réponse, en libérant la requête
