cubrid_error

(PECL CUBRID >= 8.3.1)

cubrid_error — Récupère le message de l'erreur

Description

cubrid_error ([ resource $connection ] ) : string

La fonction cubrid_error() est utilisée pour récupérer le message de l'erreur survenue lors de l'utilisation de l'API CUBRID. Habituellement, elle récupère le message de l'erreur lorsque l'API retourne FALSE.

Liste de paramètres

connection

La connexion CUBRID.

Valeurs de retour

Le message de l'erreur survenue.

Exemples

Exemple #1 Exemple avec cubrid_error()

<?php
$con = cubrid_connect('localhost', 33000, 'demodb', 'dba', '');
$req = cubrid_execute($con, "select id, name from person");
if ($req) {
    while (list ($id, $name) = cubrid_fetch($req))
    echo $id, $name;
} else {
    echo "Code de l'erreur : ", cubrid_errno($con);
    echo "Message de l'erreur : ", cubrid_error($con);
}
?>

L'exemple ci-dessus va afficher :

Code de l'erreur : -493 Message de l'erreur : Syntax: Unknown class "person". select id, [name] from person
