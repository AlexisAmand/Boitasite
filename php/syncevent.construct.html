SyncEvent::__construct

(PECL sync >= 1.0.0)

SyncEvent::__construct — Construit un nouvel objet SyncEvent

Description

public SyncEvent::__construct ([ string $name [, bool $manual = FALSE [, bool $prefire = FALSE ]]] )

Construit un objet d'événement nommé ou non.

Liste de paramètres

name

Le nom de l'événement si c'est un objet d'événement nommé.

Note:

Si le nom existe déjà, on doit être capable de l'ouvrir avec l'utilisateur courant qui exécute le processus, ou une exception sera émise avec le contenu du message d'erreur.

manual

Spécifie si l'objet d'événement doit être ou non réinitialisé manuellement.

Note:

La réinitialisation manuelle des objets d'événements permet la mise en attente des processus tant que l'objet n'est pas réinitialisé.

prefire

Spécifie s'il faut ou non pré-envoyer (le signal) à l'objet event.

Note:

N'a d'impact que si l'appel au processus/thread est le premier à créer l'objet.

Valeurs de retour

Le nouvel objet SyncEvent. Une exception est lancée si l'objet d'événement ne peut être créé ou ouvert.

Exemples

Exemple #1 Exemple avec SyncEvent::__construct()

<?php
// Dans une application web :
$event = new SyncEvent("GetAppReport");
$event->fire();

// Dans un cron :
$event = new SyncEvent("GetAppReport");
$event->wait();
?>
