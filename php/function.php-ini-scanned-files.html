php_ini_scanned_files

(PHP 4 >= 4.3.0, PHP 5, PHP 7)

php_ini_scanned_files — Retourne la liste des fichiers .ini analysés dans les dossiers de configuration supplémentaires

Description

php_ini_scanned_files ( void ) : string

php_ini_scanned_files() retourne une liste de noms de fichiers de configuration analysés après php.ini. Cette liste est au format CSV. Les dossiers examinés sont définis par une option de configuration lors de la compilation, et optionellement par une variable d'environnement lors de l'exécution : plus d'information sont disponible dans le guide d'installation.

Les fichiers de configuration retournés incluent le chemin entier.

Valeurs de retour

Retourne une chaîne de caractères où les fichiers .ini sont séparés par des virgules en cas de succès. Chaque virgule est suivie d'un retour à la ligne. Si la directive de configuration --with-config-file-scan-dir n'a pas était définit et que la variable d'environnement PHP_INI_SCAN_DIR n'est pas définit, FALSE est retourné. Si c'était définit et que le dossier était vide, une chaîne vide est retournée. Si un fichier est méconnaissable, le fichier sera quand même inclus dans la chaîne de caractères retourné mais entraînera également une erreur PHP. Cette erreur PHP sera visible à la fois au moment de la compilation et lors de l'utilisation de php_ini_scanned_files().

Exemples

Exemple #1 Un exemple de liste retournée par php_ini_scanned_files()

<?php
if ($filelist = php_ini_scanned_files()) {
    if (strlen($filelist) > 0) {
        $files = explode(',', $filelist);

        foreach ($files as $file) {
            echo "<li>" . trim($file) . "</li>\n";
        }
    }
}
?>
