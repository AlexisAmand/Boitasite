Architecture

Le plugin de multiplexage des connexions mysqlnd est implémenté sous la forme d'une extension PHP. Il est écrit en C et opère sous PHP. Lors du démarrage de l'interpréteur PHP, dans la phase d'initialisation des modules du moteur PHP, il sera enregistré comme un plugin mysqlnd pour remplacer des méthodes mysqlnd C spécifiques.

La bibliothèque mysqlnd utilise des flux PHP pour communiquer avec le serveur MySQL. Les flux PHP sont accédés par la bibliothèque mysqlnd via le module net. Le plugin de multiplexage de connexions mysqlnd agit comme un proxy pour les méthodes de la bibliothèque mysqlnd du module net pour contrôler, ouvrir et fermer les flux réseaux.

Avant d'ouvrir une connexion utilisateur à MySQL via les fonctions appropriées (mysqli, PDO_MYSQL ou ext/mysql), le plugin va rechercher une connexion réseau ouverte dans la file d'attente des connexions. Si la file d'attente contient une connexion réseau vers l'hôte spécifié par la fonction de connexion, utilisant la méthode de transport demandée (TCP/IP, Socket de domaine Unix, pipe nommé Windows), la connexion de la file est liée au gestionnaire utilisateur. Sinon, une nouvelle connexion réseau est ouverte, mise dans la file d'attente, et associée avec le gestionnaire de connexion utilisateur. De cette façon, de multiples gestionnaires utilisateurs peuvent être liés à la même connexion réseau.
