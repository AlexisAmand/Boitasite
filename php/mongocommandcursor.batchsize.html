MongoCommandCursor::batchSize

(PECL mongo >=1.5.0)

MongoCommandCursor::batchSize — Limite le nombre d'éléments retournés dans un lot

Description

public MongoCommandCursor::batchSize ( int $batchSize ) : MongoCommandCursor

Un curseur, typiquement, parcourt un lot d'objets résultants et les stocke locallement. Cette méthode définit la valeur batchSize pour configurer la quantité de documents à récupérer depuis le serveur dans un seul paquet de données.

Liste de paramètres

batchSize

Le nombre de résultats à retourner par lot. Chaque lot requiert un aller-retour au serveur.

Ceci ne peut pas écraser la limite MongoDB de données que le serveur peut retourner au client (i.e. si vous définissez la taille du lot à 1.000.000.000, MongoDB continuera de retourner 4-16MB de résultats par lot).

Valeurs de retour

Retourne ce curseur.

Exemples

Exemple #1 Exemple avec MongoCommandCursor::batchSize()

<?php
$commandCursor->batchSize(20);
?>
