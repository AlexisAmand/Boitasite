Les drapeaux d'événements

Event::READ Ce drapeau indique que l'événement devient actif lorsque le descripteur de fichier fourni (habituellement une ressource de flux ou un socket) est prêt à être lu.

Event::WRITE Ce drapeau indique que l'événement devient actif lorsque le descripteur de fichier fourni (habituellement, une ressource de flux ou un socket) est prêt à être lu.

Event::SIGNAL Ce drapeau est utilisé pour implémenter la détection de signaux. Voir la création d'un événement de type signal ci-dessous.

Event::TIMEOUT Ce drapeau indique que l'événement devient actif après l'expiration d'un délai maximal. Le drapeau Event::TIMEOUT est ignoré lors de la construction d'un événement : un délai peut être défini lors de l'ajout de l'événement, ou non. Il est défini dans l'argument $what de la fonction de rappel lorsque le délai maximal est atteint.
