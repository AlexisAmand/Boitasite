Configuration à l'exécution

Le comportement de ces fonctions est affecté par la configuration dans le fichier php.ini.

uopz Options de configuration
Nom Défaut Modifiable Historique
uopz.disable "0" PHP_INI_SYSTEM Disponible à partir de uopz 5.0.2
uopz.exit "0" PHP_INI_SYSTEM Disponible à partir de uopz 6.0.1
uopz.overloads "1" PHP_INI_SYSTEM Disponible à partir de uopz 2.0.2. Supprimé à partir de uopz 5.0.0.
Pour plus de détails sur les modes PHP_INI_*, reportez-vous à Où une directive de configuration peut être modifiée.

Voici un éclaircissement sur l'utilisation des directives de configuration.

uopz.disable bool

Si activé, uopz devrait ne plus avoir d'effet sur le moteur.

uopz.exit bool

Autoriser ou non l'exécution des codes d'opération (opcodes) de sortie. Ce paramètre peut être dérogé lors de l'exécution en appellant uopz_allow_exit().

uopz.exit bool

Active la possibilité d'utiliser uopz_overload().

Note: Lors de l'exécution avec OPcache d'activé, il est peut être nécéssaire de désactiver toutes les optimisations OPcache (opcache.optimization_level=0).
