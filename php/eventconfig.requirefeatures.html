EventConfig::requireFeatures

(PECL event >= 1.2.6-beta)

EventConfig::requireFeatures — Entre dans une fonctionalité de méthode d'événement demandée par l'application

Description

public EventConfig::requireFeatures ( int $feature ) : bool

Entre dans une fonctionalité de méthode d'événement demandée par l'application.

Liste de paramètres

feature

Masque de fonctionalités demandées. Voir les constantes EventConfig::FEATURE_*.

Valeurs de retour

Exemples

Exemple #1 Exemple avec EventConfig::requireFeatures()

<?php
$cfg = new EventConfig();

// Crée un event_base associé avec la configuration
$base = new EventBase($cfg);

// Demande de la fonctionalité FDS
if ($cfg->requireFeatures(EventConfig::FEATURE_FDS)) {
    echo "Fonctionalité FDS demandée\n";

    $base = new EventBase($cfg);
    ($base->getFeatures() & EventConfig::FEATURE_FDS)
        and print("FDS - types arbitraire de descripteur de fichier et non pas juste des sockets\n");
}
?>

L'exemple ci-dessus va afficher quelque chose de similaire à :

Fonctionalité FDS demandée
FDS - types arbitraire de descripteur de fichier et non pas juste des sockets
