Pré-requis

L'extension OCI8 1.4 est incluse avec PHP 5.3, PHP 5.4 et PHP 5.5. Elle est également disponible depuis » PECL. Une version OCI8 2.0 plus récente est disponible uniquement depuis » PECL. OCI8 1.4 requiert les bibliothèques clientes Oracle 12c, 11g, 10g ou 9iR2 et peut s'installer sur PHP 4.3.9 et suivants. OCI8 2.0 requiert les bibliothèques clientes Oracle 12c, 11g ou 10g et peut s'installer sur PHP 5.2 et suivants.

Si la base de données Oracle est sur la même machine que PHP, le logiciel de base de données contient déjà les bibliothèques nécessaires. Lorsque PHP est sur une machine différente, utilisez les bibliothèques gratuites » Oracle Instant Client.

Pour utiliser Oracle Instant Client, installez le fichier compressé basic ou basiclite de Oracle Instant Client ou le paquet RPM. Lorsque vous compilez PHP depuis ces sources, installez également le fichier compressé sdk ou le paquet RPM devel.

Vous devez exécuter PHP avec la même version ou une version plus récente des bibliothèques clients Oracle utilisées pour construire OCI8.

Sous Windows, la bibliothèque DLL php_oci8 requiert les bibliothèques clientes depuis la version 10gR2 ou supérieure. En PHP 5.3, incluant PHP 5.3.5, la bibliothèque DLL php_oci8_11g requiert les bibliothèques clientes Oracle 11gR1 ou supérieur. A partir de PHP 5.3.6, la bibliothèque DLL php_oci8_11g requiert les bibliothèques clientes Oracle 11gR2 ou supérieur. Depuis PHP 5.3.6, la bibliothèque DLL php_oci8_11g requiert les bibliothèques clientes Oracle 11gR2 ou supérieures. La bibliothèque DLL php_oci8_12c depuis PECL requiert les bibliothèques clientes Oracle 12cR1. Suivant la version du client Instant, vous pourriez également avoir besoin des bibliothèques mfc71.dll et msvcr71.dll.

Note:

Si OCI8 utilise les bibliothèques clientes 9iR2, alors PHP peut se connecter aux bases de données Oracle 8i, 9iR2, 10g ou 11g. Si OCI8 utilise les bibliothèques clientes 10gR2, la base de données peut être 9iR2, 10g, 11g ou 12c. Si OCI8 utilise les bibliothèques clientes 11g, la base de données peut être 9iR2, 10g, 11g ou 12c. Si OCI8 utilise les bibliothèques clientes 12c, la base de données peut être 10gR2, 11g ou 12c.

Note:

Le support de toutes les fonctionnalités OCI8 n'est disponible que lors de l'utilisation des versions les plus récentes des bibliothèques clientes Oracle mais aussi de la base de données la plus récente.
