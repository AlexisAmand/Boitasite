Mongo::poolDebug

(PECL mongo >=1.2.0)

Mongo::poolDebug — Retourne des informations sur les pools de connexion

Description

public Mongo::poolDebug ( void ) : array
Avertissement

Cette fonctionnalité est devenue OBSOLÈTE depuis la version 1.2.3. Nous vous encourageons vivement à ne plus l'utiliser. Utilisez la méthode MongoPool::info() à la place.

Retourne un tableau contenant des informations sur les pools de connexion.

Liste de paramètres

Cette fonction ne contient aucun paramètre.

Valeurs de retour

Chaque pool de connexion possède un identifiant, qui commence par l'hôte. Pour chaque pool, cette fonction fournit les champs suivants:

in use

Nombre de connexions utilisées actuellement par les instances MongoClient.

in pool

Le nombre de connexions actuellement dans le pool (non utilisées).

remaining

Le nombre de connexions qui pourraient être créées dans ce pool. Par exemple, supposons un pool avec 5 connexions restantes et 3 utilisées. Nous pourrions créer 8 instances de MongoClient avant d'arriver à saturer le pool (supposant que toutes les instances ont réussi MongoClient).

Un nombre négatif signifie que le pool est illimité en connexions.

Avant que le pool ne soit crée, vous pouvez changer le nombre maximum de connexions en appelant Mongo::setPoolSize(). Une fois le pool crée, vous ne pouvez plus changer sa taille.

timeout

Le timeout pour le socket de connexion de ce pool. C'est le temps pendant lequel les connexions dans ce pool vont attendre pour se connecter à un serveur.
