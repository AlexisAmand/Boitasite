cubrid_disconnect

(PECL CUBRID >= 8.3.0)

cubrid_disconnect — Ferme la connexion à la base de données

Description

cubrid_disconnect ([ resource $conn_identifier ] ) : bool

La fonction cubrid_disconnect() ferme le gestionnaire de connexion et se déconnecte du serveur. S'il existe des gestionnaires de requêtes non fermés, à ce moment là, ils le seront. Cette fonction est similaire à la fonction cubrid_close().

Liste de paramètres

conn_identifier

Identifiant de connexion.

Valeurs de retour

TRUE, en cas de succès.

FALSE, si une erreur survient.

Exemples

Exemple #1 Exemple avec cubrid_disconnect()

<?php
$con = cubrid_connect ("localhost", 33000, "demodb");
if ($con) {
   echo "Connexion réussie";
   
   $req = cubrid_execute( $con, "create table person(id int,name char(10))");
   if ($req) {
      cubrid_close_request($req);
      cubrid_commit($con);
   } else {
      cubrid_rollback($con);
   }
   
   $req = cubrid_execute( $con, "insert into person values(1,'James')");
   if ($req) {
      cubrid_close_request($req);
      cubrid_commit($con);
   } else {
      cubrid_rollback($con);
   }
   cubrid_disconnect($con);
}
?>
