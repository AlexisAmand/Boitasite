cubrid_get_query_timeout

(PECL CUBRID >= 8.4.1)

cubrid_get_query_timeout — Récupère la valeur du délai d'attente maximal d'une requête

Description

cubrid_get_query_timeout ( resource $req_identifier ) : int

La fonction cubrid_get_query_timeout() est utilisée pour récupérer le délai d'attente maximal d'une requête.

Liste de paramètres

req_identifier

Un identifiant de requête.

Valeurs de retour

Retourne la valeur du délai d'attente maximal pour la requête en cas de succès, en millisecondes.

Retourne FALSE si une erreur survient.

Exemples

Exemple #1 Exemple avec cubrid_get_query_timeout()

<?php

$host = "localhost";
$port = 33000;
$db = "demodb";

$conn =
cubrid_connect_with_url("CUBRID:$host:$port:$db:::?login_timeout=50000&query_timeout=5000&disconnect_on_query_timeout=yes");

$req = cubrid_prepare($conn, "SELECT * FROM code");

$timeout = cubrid_get_query_timeout($req);
var_dump($timeout);

cubrid_set_query_timeout($req, 1000);
$timeout = cubrid_get_query_timeout($req);
var_dump($timeout);

cubrid_close($conn);
?>

L'exemple ci-dessus va afficher :

int(5000)
int(1000)
