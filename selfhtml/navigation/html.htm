Remarque

Le sommaire de référence HTML n'est pas conçu pour les débutants! Le sommaire de référence HTML s'adresse aux utilisateurs qui s'y connaissent déjà en HTML et qui désirent avoir lors de leur travail quotidien une vue générale de la syntaxe et des éléments du langage importants dans la pratique (Le sommaire de référence ne prétend pas être exhaustif!). Le sommaire de référence est tout indiqué comme aide à la navigation. À partir des courts descriptifs, des liens du type  description mènent aux explications détaillées dans la documentation.

Mentions d'entête HTML

Mention Meta:
auteur

 <meta name="author" content="Name">
Noter pour Name le nom de l'auteur.
 description
 

Mention Meta:
description

 <meta name="description" content="Text">
Noter pour Text un descriptif sommaire du contenu.
 description
 

Mention Meta:
mots clés

 <meta name="keywords" content="mot, mot, mot">
Noter pour chaque mot un mot-clé sur le contenu . Leur nombre est variable. Séparer les mots clés par des virgules.
 description
 

Mention Meta:
date et heure

 <meta name="date" content="yyyy-MM-ddThh:mm:ss+hh:mm">
yyyy = année, MM = Mois, dd = jour, T = lettre invariable, hh = heures, mm = minutes, ss = secondes, +/- = variation en heures et en minutes par rapport à l'heure de Greenwich.
 description
 

Mention Meta:
Robots

 <meta name="robots" content="index|noindex|follow|nofollow">
Instruction aux moteurs de recherche:
index = lecture permise ,
noindex = lecture non permise,
follow = suivre les liens permis,
nofollow = suivre les liens non permis.
 description
 

Mention Meta:
jeu de caractères

 <meta http-equiv="content-type" content="type Mime; charset=jeu de caractères">
Noter pour type Mime quelque chose comme text/html et pour jeu de caractères quelque chose comme ISO-8859-1.
 description
 

Mention Meta:
langage de style

 <meta http-equiv="Content-Style-Type" content="type Mime">
Noter pour type Mime quelque chose comme text/css.
 description
 

Mention Meta:
langage script

 <meta http-equiv="Content-Script-Type" content="type Mime">
Noter pour type Mime quelque chose comme text/javascript.
 description
 

Mention Meta:
adresse d'origine

 <meta http-equiv="expires" content="0">
Mentionner la date à partir de laquelle il faut charger de l'adresse originale. 0 signifie: toujours charger de l'adresse d'origine (ignorer le serveur Proxy).
 description
 

Mention Meta:
Faire suivre
automatiquement

<meta http-equiv="refresh" content="0; URL=URI">
Mentionner quand et quelle adresse doit être chargée automatiquement. 0 signifie: tout de suite (mention exprime le délai en secondes).
Pour URI mentionner une adresse Web ou une cible avec ou sans chemin qui doit être chargée automatiquement après le temps mentionné.
 description
 

Adresse de base

 <base href="URI">
Pour URI mentionner une adresse Web ou une cible avec ou sans chemin qui doit servir de base pour un adressage relatif.
 description
 

Fenêtre cible de base

 <base target="Nom">
fenêtre cible standard pour les liens pour des cadres, pour les liens proprement-dits, aucune mention n'est plus alors nécessaire pour target=. Pour Nom mentionner le nom de fenêtre désiré.

Relations
logiques

 <link rel="relation" title="Titre"href="URI">
<link rev="relation" title="Titre" href="URI">
rel= équivaut à relation, rev= à relation de retour.
Pour relation Pour les deux attributs les mentions suivantes sont possibles:
contents = index du contenu
chapter = chapitre
section = section
subsection = sous-section
index = index des mots clés
glossary = glossaire
appendix = annexe
copyright = remarques sur les droits
next = page suivante
prev = page précédente
start = page de début
help = page d'aide
bookmark = point d'orientation générale
alternate = autre mode de lecture
stylesheet = incorporer un fichier de feuilles de style externe
Pour Titre noter un texte pour l'inscription pour le cas où celle-ci apparaît.
Pour URI mentionner une adresse Web ou une cible avec ou sans chemin qui doit être reliée.

prédéfinir des
colonnes

 <table >
<colgroup>
<col>
<col width="largeur" span="nombre">

<tr>
<td>cellule</td>
<td>cellule</td>
</tr>

Avec des repères <col> prédéfinir autant de colonnes que le tableau compte. Groupez les avec des éléments colgroup.

span = Mention sur la largeur s'appliquant à autant de colonnes qui suivent que mentionné.
Pour nombre mentionner un nombre comme par exemple 3 pour le nombre de colonnes auxquelles les mentions doivent s'appliquer.

Liens (Links)

Verweis definieren

 <a href="URI">texte du lien</a>
Pour URI mentionner une adresse Web ou la cible avec ou sans chemin du fichier ou de la source devant être relié.
URI peut être:
une ancre dans le même fichier (par exemple #nom d'ancre)
un autre fichier (par exemple fichier.htm)
un autre fichier dans un autre répertoire (par exemple ../répertoire/fichier.htm)
une ancre dans un autre fichier (par exemple fichier.htm#nom d'ancre)
une adresse WWW (http://...)
une adresse Gopher (gopher://...)
une adresse FTP (ftp://...)
une adresse Telnet (telnet://...)
une adresse de groupe de nouvelles (news:...)
une adresse locale absolue (file://...)
 description
 

Définir une ancre

 <a name="nom d'ancre">[contenu]</a>
Pour nom d'ancre noter un nom. Ni espace, ni accent, comme premier caractère une lettre à part cela chiffres et tirets de soulignement (_), trait d'union (-), deux points (:) ou point (.).
 description
 

Fenêtre-cible
pour les liens

 <a href="URI" target="Fenêtre-cible">texte du lien</a>
Pour Fenêtre-cible mentionner le nom d'une fenêtre-cadre définie ou l'un des noms réservés suivants:
_blank = ouvrir le lien dans une nouvelle fenêtre,
_self = ouvrir le lien dans la même fenêtre,
_parent = terminer le jeu de cadres actuel lors de l'exécution du lien,
_top = terminer tous les jeux de cadres lors de l'exécution du lien.
 description

Pour alignement noter une des valeurs suivantes:
top = en haut avec du texte avant ou après,
middle = au milieu par rapport au texte avant ou après,
bottom = en bas avec du texte avant ou après.
left = aligner l'applet à gauche (les contenus qui suivent passent à droite),
right = aligner l'applet à droite (les contenus qui suivent passent à gauche),
Pour gauchedroite noter un nombre comme par exemple 10 pour déterminer l'espace en pixels entre l'applet et son environnement à gauche et à droite.
Pour hautbas noter un nombre comme par exemple 10 pour déterminer l'espace en pixels entre l'applet et son environnement en haut et en bas.
