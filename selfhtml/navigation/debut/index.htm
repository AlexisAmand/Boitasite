De quoi s'agit-il?

Ce que vous avez en ce moment sur votre écran, est une documentation assez importante dans laquelle il est question de savoir comment on établit des pages pour l'Internet ou pour l'Intranet interne à l'entreprise. Faire de telles pages n'est pas quelque chose de sorcier que seuls quelques fanas d'informatique maîtrisent. Si vous avez bonne volonté et envie, vous pouvez acquérir le "savoir faire". Pour ce faire, vous pouvez commencer petit et tester un peu. Si vous prenez plaisir à la chose vous pouvez au fur et à mesure étendre vos connaissances.

Partant de cette page, vous pouvez faire des "tours" guidés à travers la présente documentation. Ces tours doivent vous faciliter la mise en route. Pour tous ces tours apparaît une fenêtre coupée en deux. A gauche, vous pouvez choisir les liens qui font partie du tour. A droite sont montrées les pages que vous appelez en cliquant sur les liens de la fenêtre de gauche. Il s'agit de pages normales qui font partie de cette documentation et que vous pouvez également atteindre par la page d'accueil de SELFHTML.

Si vous n'avez encore jamais feuilleté ni visité SELFHTML, vous devriez d'abord prendre connaissance des "Conditions générales" de cette documentation. N'ayez pas peur il n'y a que deux pages. Pour ce faire, appelez le tour 1:

  

Que dois-je d'abord savoir?

SELFHTML n'est pas une introduction au mode d'emploi de votre ordinateur. Vous n'y trouverez pas les connaissances de base. Vous devez donc par exemple savoir ce que sont un fichier et un dossier ou répertoire et qu'il y a des formes très différentes de fichiers. Vous devriez avoir déjà travaillé avec différents programmes connaître un ou plusieurs d'entre eux un peu mieux et savoir qu'il y a différents programmes pour différentes tâches.

Votre navigateur Web, avec lequel vous lisez ceci est lui aussi un programme sur votre ordinateur ou sur votre réseau local. Des données qui sont montrées dans la fenêtre de votre navigateur sont des fichiers qui sont enregistrés quelque part. Le lieu où ils sont enregistrés peut être sur un ordinateur éloigné sur Internet dans l'Intranet de votre firme ou organisation, sur un CD-ROM que vous venez d'introduire dans le lecteur ou aussi sur disque dur de votre ordinateur.

Pour bien comprendre SELFHTML vous serez avantagé si vous avez déjà souvent "surfé" sur Internet. Avantagé en ce sens que vous aurez déjà vu diverses pages Internet et que vous aurez donc plus de possibilités de comparaison et de stimulation pour vos propres idées. SELFHTML s'adresse en tous cas à ceux qui désirent publier eux même des pages sur Internet ou sur l'Intranet de leur entreprise. Les "pages Internet", "pages WWW", "pages Web" etc. sont d'ailleurs différentes expressions pour la même chose - il s'agit simplement de certaines sortes de formats de données qu'un navigateur Web peut afficher dans sa fenêtre.

Avant que vous attaquiez le vif du sujet, vous devriez faire le point. Je veux dire par là que vous devriez d'abord acquérir une vision globale des thèmes débattus ici. Pour obtenir une vision globale de l'Internet, du World Wide Web et de la façon de publier ses propres pages Internet, appelez maintenant le tour 2:

Pour la prochaine étape, vous devez savoir quelles technologies existent pour créer vos pages eb et quelles sont les spécificités des différents langages. Appelez le tour 3 pour apprendre à connaître les différents langages et technologies qui jouent un rôle pour la création de pages Internet.

Avant que vous ne commenciez à concevoir vos propres pages, vous aurez encore quelques questions concernant les logiciels nécessaires ou quelques questions comme par exemple comment placer les pages créées sur la toile. Appelez pour cela le tour 4.

  

Dans quel ordre dois-je apprendre?

Il y a toujours plusieurs façons et apprendre est souvent avant tout quelque chose d'individuel. Finalement une méthode vous sera ici proposée pour pouvoir faire vos premiers pas en HTML. Pour cela il vous faut savoir comment vous pouvez changer entre plusieurs applications ouvertes sur votre ordinateur. Ensuite vous passerez souvent de l'éditeur à deux instances ouvertes de votre navigateur et inversement.

Pour vos premiers pas en HTML vous pouvez maintenant appeler votre éditeur HTML. Au cas ou vous n'en avez encore aucun, vous pouvez appeler aussi un éditeur de texte qui ne soit pas si possible un programme de traitement de texte comme Word ou similaire. Les utilisateurs de Windows peuvent par exemple utiliser le bloc notes qui fait partie de Windows. Pour pouvoir travailler de façon plus efficace vous devriez de toutes façons vous procurer à terme un véritable éditeur HTML.

Laissez le navigateur ouvert avec ce document. De cette façon vous avez toujours l'aide disponible.

Créez avec l'éditeur un nouveau fichier vide et enregistrez-le sous un nom de fichier (*.htm ou *.html) par exemple premier.htm. Rechangez ensuite pour afficher ce document.

Ouvrez maintenant une seconde instance de votre navigateur Web (dans la première instance c'est bien sûr ce document qui est affiché et il doit continuer à l'être). Le menu de commande pour ouvrir une deuxième instance du navigateur peut être trouvée normalement dans le menu fichier/file du navigateur. Si le fichier HTML créé est encore vide, le navigateur affiche à l'ouverture de ce fichier une fenêtre d'affichage vide.

Maintenant vous avez votre environnement de travail: Dans l'éditeur ouvert vous pouvez composer votre premier fichier HTML. Avec l'instance du navigateur avec laquelle vous avez ouvert ce fichier HTML vous pouvez contrôler à tout moment ce que vous avez "fabriqué" dans l'éditeur. Pour contrôler vos entrées à tout moment choisissez dans l'éditeur la commande pour sauvegarder le fichier HTML. Passez ensuite à l'instance du navigateur dans laquelle le fichier HTML est affiché. Pour afficher le fichier HTML modifié choisissez dans le navigateur la commande recharger la page en cours pour actualiser la page Web affichée.

De temps à autre vous devrez changer pour cette instance-ci du navigateur. Car c'est ici que vous trouvez la description des commandes HTML que vous pouvez entrer dans l'éditeur. Le tour 5 qui suit vous mène aux plus importantes connaissances en HTML. Encore un tuyau: sélectionnez les exemples en code-source HTML qui sont présentés au fil du tour ici dans le navigateur avec la souris. Copiez le texte sélectionné dans le presse-papiers de votre ordinateur puis collez-le dans votre éditeur HTML .

  

Comment devenir spécialiste?

Le plus souvent vous apprenez quand vous réfléchissez à un projet que vous voulez réaliser et que vous voulez aussi publier réellement sur le Web. �a peut être une page d'accueil personnelle ou commerciale, un projet de passionné pour quelqu'un ou quelque chose, un projet de fan comme la présentation d'une collection la présentation de votre collectivité de quartier, de votre école de votre club etc... Réfléchissez au contenu que vous désirez y mettre comment vous pouvez répartir ce contenu sur plusieurs pages et comment vous pouvez relier ces pages entre elles de façon à ce que l'utilisateur puisse bien naviguer dans le projet.

Apprenez à connaître pendant ce travail les autres éléments de  HTML. Vous pouvez aussi tranquillement suivre l'un des nombreux liens dans la documentation HTML qui mènent dans cette documentation aux  feuilles de style CSS.

Le dernier Tour 6 vous initie à d'autres thèmes qui traitent de thèmes avancés.

Les pages que vous allez connaître ou avez connues au cours de ces 6 tours ne sont bien sûr pas les seules dans SELFHTML. Mais vous l'aurez déjà remarqué vous-même. Après les tours que nous vous avons proposés ici vous êtes, je l'espère en mesure de composer de façon autonome des pages Internet et de consulter SELFHTML quand vous cherchez une solution à un problème. La route pour devenir spécialiste est longue et ne se parcourt pas en quelques heuresni même en quelques jours. Mais vous vous êtes aventuré sur un terrain où il y a tous les jours quelque chose de nouveau à apprendre. Bonne Chance!
