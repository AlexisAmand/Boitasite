Conventions pour éditer en XML

Comme en HTML le code source XML est du texte clair. Pour que vos données en code source restent claires, vous pouvez utiliser la tabulation ou les passages à la ligne. Pour le passage à la ligne les caractères de contrôle pour le retour du chariot (CR - carriage return) et l'avance d'une ligne (LF - line feed) sont permis. Ce qui signifie que vous pouvez travailler sans problème sur des fichiers XML avec des éditeurs de texte courants sous Windows MS, les dérivés Unix, Macintosh etc...

Plusieurs espaces ou lignes vides à la suite sont résumées en un seul espace tant que vous ne mentionnez rien d'autre pour un élément dans la DTD. Pour forcer l'insertion de plusieurs espaces dans un texte, il vous faut utiliser l'espace protégé (&#160;). l'entrée de &#160;&#160;&#160; force par exemple l'insertion de 3 espaces à la suite. Ne perdez pas de vue que, à la différence de HTML, vous ne pouvez pas utiliser ici le signe nommé &nbsp;, aussi longtemps qu'aucune  Entité (Entity) n'a été définie à cet effet dans la DTD.

Pour le formatage de données XML, vous devez utiliser un langage de feuilles de style (voir  XSL ou bien CSS - formatage d'éléments XML).

  

Conventions de noms de fichiers

Pour les documents XML, il n'y a pas de conventions de noms de fichiers prescrites. Si l'on considère l'utilisation sur le WWW où des serveurs Web et des navigateurs appelants doivent sans cesse se mettre d'accord sur le format de données à transmettre (le  type Mime), il est cependant conseillé de s'en tenir aux deux règles suivantes:

1. Les documents XML sans référence DTD déterminée doivent recevoir l'extension .xml. Cette extension est en principe portée par défaut pour le type Mime text/xml s'appliquant aux fichiers XML.

2. Les documents XML qui se réfèrent à des DTD publiques, doivent utiliser l'extension de fichier que les documents d'exemple publics utilisent. Ainsi par exemple des fichiers WML doivent-ils par exemple utiliser l'extension .wml.
