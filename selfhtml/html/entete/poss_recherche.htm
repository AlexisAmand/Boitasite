 Définir un fichier comme champ de recherche

Aujourd'hui, cette commande n'a plus d'utilité dans la pratique étant donné que les navigateurs WWW modernes permettent des recherches de texte rapides et confortables dans un fichier HTML affiché . L'élément a également été classé entre-temps comme en cours d'abandon et doit dans le futur quitter le standard HTML. C'est pourquoi les informations qui suivent ont davantage un caractère "historique".

Un fichier HTML peut être un champ de recherche. Ce qui signifie que son contenu a été indexé sur l'ordinateur serveur avec tous les mots qu'il contient. Il n'est cependant judicieux de définir un fichier HTML comme champ de recherches que si son contenu se trouve effectivement sous forme indexée, c'est à dire dans un fichier index de recherches sur l'ordinateur serveur.

Pour un fichier HTML défini comme champ de recherche apparaît à l'affichage sur le navigateur WWW et au début du fichier un champ de saisie qui permet à l'utilisateur d'entrer un terme recherché.

Exemple:

<head>
<isindex prompt="Entrez le terme recherch�: ">

Explication:

Avec isindex vous déclarez un fichier HTML comme champ de recherche. Avec l'attribut prompt= vous déterminez un texte d'invite. Le texte doit figurer entre guillemets (prompt = invite).
