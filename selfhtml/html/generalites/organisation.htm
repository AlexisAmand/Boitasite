Écrire de l'hypertexte - répartir l'information

HTML sont les initiales de Hyper Texte Markup Language. Le langage est conçu pour une répartition de l'information orientée sur l'hypertexte. L'utilisateur doit naviguer dans des informations qui l'intéressent. Si vous vous intéressez à ce thème, vous devez lire dans l'introduction de ce document le sous-chapitre consacré à l'  hypertexte.

Il est pour cela nécessaire que l'information proposée soit divisée en petites unités unités indépendantes reliées entre elles de façon reconnaissable et pouvant être suivie.

Si par exemple vous voulez présenter votre club de football sur le WWW il vous faut faire la page d'accueil habituelle avec des liens qui mènent aux sous-pages. Les sous-pages doivent elles aussi de la même façon comporter des liens de retour à la page d'accueil. Dans cet exemple vous pouvez consacrer une sous-page à l'histoire, aux articles sur le club, aux résultats actuels et aux donations aux joueurs actuels. La page pour les joueurs pourrait contenir encore d'autres liens menant à des pages où les joueurs sont présentés individuellement. Pour de telles sous-pages un lien de retour à la page immédiatement supérieure ainsi qu'à la page d'accueil serait élégant. Des liens croisés sont eux aussi importants. Si un joueur est déjà là depuis longtemps on peut certainement imaginer un lien croisé à un événement important de l'histoire du club. Pour les explications sur les résultats on peut là aussi imaginer des liens croisés menant aux meilleurs gardiens de but etc..
Ainsi apparaît un réseau d'informations distinctes. Des liaisons supplémentaires devraient compléter les possibilités de navigation. Il est ainsi judicieux dans de nombreux cas de proposer des visites guidées à savoir une suite de pages que l'utilisateur peut feuilleter confortablement.
Peu importe le projet auquel vous vous attaquez: vous devez toujours avoir en tête l'esquisse qui vient de vous être faite pour la structure de l'information.

Ce qui est désigné ici comme "page" ou "sous-page" devrait être dans un projet WWW un fichier HTML séparé. Ne vous effrayez pas du nombre des fichiers existants. La structure propre de l'information doit être le premier commandement.

La technique de la relation entre les fichiers du projet est décrite dans la partie traitant des  liens. D'autres tuyaux pour la conception de votre projet WWW vous sont donnés dans la partie traitant de la  planification de projets Web.

  

Organisation des fichiers

Pour chaque projet plus important vous vous rendrez vite compte du nombre de fichiers HTML graphiques et autres qui peuvent s'amasser. Pour continuer à superviser l'ensemble vous devriez soit vous trouver un système logique de noms de fichiers soit vous créer des sous répertoires pour les fichiers concernant un thème donné par exemple ou pour les graphiques etc... Si vous voulez publier votre projet sur le WWW, vous devez éclaircir au préalable dans le cas où vous utilisez des sous répertoires si vous avez le droit ou la possibilité de créer des sous répertoires sur le serveur où votre projet sera sauvegardé. Car dans ce cas, il vous faudra avoir la même arborescence sur l'ordinateur serveur que celle que vous avez choisie en établissant votre projet localement afin que tous les liens ou références fonctionnent.
