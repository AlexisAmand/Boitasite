Noms de fichiers du point de vue de l'ordinateur serveur

La plupart des ordinateurs serveurs sur le WWW aujourd'hui sont des ordinateurs avec système d'exploitation Unix ou dérivé. Unix différencie sévèrement les majuscules et les minuscules dans les noms de fichiers. Celui qui établit son projet WWW dans un environnement DOS-/Windows où cette distinction n'est pas faite, peut faire d'amères expériences si subitement les  liens et  références de graphiques qui fonctionnent localement ne fonctionnent plus après avoir été chargés sur l'ordinateur serveur. La raison en est dans de tels cas que par exemple la cible des liens ou des références de graphiques dans le fichier HTML est écrite en minuscules alors le système DOS-/Windows a transmis au serveur des noms de fichiers écrits en majuscules .

Si vous travaillez avec Windows 3.x ou DOS 6.x et plus bas, il ne vous reste plus dans un tel cas comme possibilité qu'à renommer les fichiers sur le serveur comme il sont notés dans les liens et références de graphique et ce à l'aide de commandes FTP. Si vous travaillez avec un système qui différencie majuscules et minuscules à l'apparence mais pas en interne (i.e. Windows 95) il est plus sûr de créer des noms de fichiers composés exclusivement de minuscules et de noter de la même façon les liens et les références de graphiques en minuscules.

  

Noms de fichiers du point de vue de la compatibilité

Si vous n'établissez votre projet Web que pour être employé exclusivement sur le Web vous devez faire attention à ce que les fichiers soient acceptés par l'ordinateur serveur. Dans des systèmes Unix modernes comme les utilisent la plupart des serveurs WWW les noms de fichiers peuvent comporter jusqu'à 256 signes. Évitez dans tous les cas les signes avec accent et caractères spéciaux dans les noms de fichiers. Le tiret de soulignement "_"est par contre un caractère spécial accepté. D'autres signes de ponctuation sont à éviter. Point d'interrogation "?", astérisque "*", signe égal "=" ne peuvent en aucun cas être utilisés dans les noms de fichier. Renoncez à tout prix à l'espacement.

Si par contre vous désirez utiliser votre projet autrement par exemple pour être téléchargé et lu hors connexion (comme le présent document) ou sur CD-ROM vous devez veiller aux normes strictes des noms de fichier du MS-DOS afin que ces utilisateurs ne rencontrent pas de problèmes avec l'affichage du projet. Ces règles précisent qu'1 point peut séparer le nom du fichier sur 8 caractères de son extension sur 3 caractères).

  

Extensions de fichiers

Ici la règle est la suivante: respectez absolument les extensions de fichiers habituelles. Des navigateurs WWW modernes reconnaissent certains types de fichiers également à leur contenu mais du point de vue de l'absence d'erreurs vous devriez toujours utiliser les extensions habituelles.

  • les fichiers HTML ont toujours l'extension .html ou .htm.
  • les fichiers HTML avec des  Server Side Includes ont l'extension .shtml ou .shtm.
  • les fichiers HTML avec des parties  PHP ont l'extension .php.
  • les fichiers HTML avec des parties  ASP ont l'extension .asp.
  

Fichier par défaut sur les serveurs Web

La plupart des serveurs WWW d'aujourd'hui prévoient un nom réservé au fichier HTML qui servira de "page d'accueil". Généralement, il s'agit du nom index.html ou index.htm, quelquefois aussi welcome.html ou welcome.htm ou default.html voire default.htm. Chez quelques fournisseurs vous pouvez vous-même donner un nom de fichier au choix comme nom de fichier par défaut. Et si vous avez vous même accès à la configuration du serveur Web, vous pouvez y régler le nom de fichier par défaut.

L'avantage d'un fichier par défaut consiste à ce que le nom du fichier HTML n'a plus besoin d'être noté: l'adresse Web devient ainsi plus courte. Beaucoup d'adresses sont simplement http://www.xy.com/ ou http://www.xy.com/repertoire/ - sans données complémentaires d'un fichier HTML déterminé. Le fait qu'un fichier déterminé soit pourtant chargé lors de l'appel d'une telle adresse tient à ce que le serveur WWW a sauvegardé ce fichier en tant que nom de fichier par défaut .

Renseignez vous auprès de votre fournisseur d'accès s'il préconise un nom de fichier par défaut pour des répertoires et si oui lequel. Établissez ensuite votre projet de telle façon que la page d'accueil porte ce nom et que les pages subalternes désignent aussi ce fichier dans leur lien de retour en arrière.
