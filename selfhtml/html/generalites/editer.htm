Règles lors de la saisie de texte

Si vous utilisez un éditeur de texte  éditeur HTML ou un éditeur orienté code source pour établir vos fichiers HTML vous devez respecter les règles suivantes:

  • Dans un nouveau fichier notez toujours en premier lieu l' ossature d'un fichier HTML.
  • Veillez lors de la saisie de texte aux règles régissant le masquage pour les  Alphabets, caractères spéciaux et caractères HTML spécifiques.
  • Passez à la ligne et insérez des lignes vides de façon à garder une meilleure vue d'ensemble dans votre texte source. Pensez cependant aussi à ce que les navigateurs WWW n'affichent pas les césures et les changements de paragraphe comme vous les entrez dans le code source. Pour des passages à la ligne forcés ou des changements de paragraphes devant être vus dans le navigateur WWW, il vous faut entrer les commandes HTML correspondantes par exemple celles pour les  paragraphes (changement de) ou les  césures. Si pour une raison particulière vous désirez que votre texte soit affiché sur le navigateur comme vous le saisissez (avec tous les retraits passages à la ligne, césures etc...) , vous pouvez utiliser la commande HTML pour du  texte pré formaté.
  • Attention: le navigateur WWW n'interprète pas non plus les tabulations. Une tabulation entrée dans le code source devient un espace à l'affichage dans le navigateur WWW. Les césures, les tabulations et les simples espacements forment en HTML la classe de ce qu'on appelle les "white space" (= espace blanc). Les navigateurs transforment en règle générale les tabulations, les signes de passage à la ligne dans l'éditeur, en espace dans le texte HTML. Plusieurs espaces qui se suivent sont ignorés par le navigateur WWW et interprétés comme un seul. Pour forcer plusieurs espaces utilisez, à la place de la barre d'espacement, le caractère de contrôle &#160; ( espace protégé) aussi souvent à la suite l'un de l'autre que l'espace désiré doit être grand.
